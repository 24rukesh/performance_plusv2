<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → st_db.save_analysis | The "Analysis label" text_input value flows into the label column. It is stored as TEXT and rendered via st.markdown in the sidebar. |
| st_db.load_analysis → session state | Payload from Postgres JSONB is deserialized and used to set session state keys directly. Pydantic model_validate provides schema validation on the analysis_result portion. DataFrames are reconstructed via pd.DataFrame() from records. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-08 | Injection | Analysis label text_input | accept | Label stored in TEXT column via psycopg2 parameterized query (%s). No SQL injection risk. Rendered as sidebar markdown (no unsafe_allow_html=True). |
| T-12-09 | Tampering | Load restores session state without validation of campaign_agg/merged_df | mitigate | analysis_result is validated via AnalysisResult.model_validate() — schema violations raise pydantic.ValidationError immediately. campaign_agg and merged_df are reconstructed via pd.DataFrame(records) — malformed records surface as empty DataFrames, not code execution. |
| T-12-10 | Denial of Service | st_db calls without try/except | mitigate | All four st_db calls (save, list, load, delete) in app.py are wrapped in try/except psycopg2.OperationalError. A dead Postgres connection degrades gracefully — Save shows st.error, sidebar shows st.sidebar.warning — app does not crash. |
| T-12-11 | Information Disclosure | Past Analyses sidebar shows all runs | accept | No multi-user isolation in Phase 12 (single-user self-hosted app). Multi-tenant isolation is explicitly deferred to v4+ per CONTEXT.md deferred ideas. |
| T-12-SC | Tampering | pip/cargo installs | mitigate | No new packages installed in this plan. psycopg2 already in pyproject.toml. |
</threat_model>
