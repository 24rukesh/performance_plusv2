    tests/test_evals.py lines 56-89 (the two fallback tests to harden):

      @pytest.mark.parametrize("exc_class", [
          openai.RateLimitError,
          openai.AuthenticationError,
          openai.APIConnectionError,
      ])
      def test_openai_error_triggers_fallback(exc_class, monkeypatch):
          mock_st = MagicMock()
          mock_st.session_state = {}
          monkeypatch.setitem(sys.modules, "streamlit", mock_st)
          exc = _make_openai_exc(exc_class)
          with patch("llm._call_llm", side_effect=exc):
              result = run_analysis(_minimal_agg_df())
          assert isinstance(result, AnalysisResult), ...
          assert mock_st.session_state.get("api_fallback_active") is True, ...


      def test_value_error_does_not_trigger_fallback(monkeypatch):
          mock_st = MagicMock()
          mock_st.session_state = {}
          monkeypatch.setitem(sys.modules, "streamlit", mock_st)
          with patch("llm._call_llm", side_effect=ValueError("bad input")):
              with pytest.raises(ValueError, match="bad input"):
                  run_analysis(_minimal_agg_df())
          assert mock_st.session_state.get("api_fallback_active") is not True

    The problem: if DEMO_MODE=1 is set in the test environment, run_analysis() returns early from its
    DEMO_MODE guard before _call_llm is ever called. The patched side_effect is never triggered,
    api_fallback_active is never set, and test_openai_error_triggers_fallback fails on the
    api_fallback_active assertion.

    The fix for BOTH tests: add these two lines immediately after the `mock_st` setup block and
    before `exc = _make_openai_exc(...)` (or before the `with patch(...)` block):
      monkeypatch.delenv("DEMO_MODE", raising=False)
      monkeypatch.setattr("llm.client", MagicMock())

    The `setattr("llm.client", MagicMock())` ensures that even in an environment where llm.client
    is None (due to missing OPENAI_API_KEY), the DEMO_MODE fallback at the top of run_analysis()
    is not triggered by a None-client guard. It does not affect the _call_llm patch because
    _call_llm is patched separately and raises before client is used.

    Do not modify test_value_error_does_not_trigger_fallback's patch or assertion logic — only
    add the two env guard lines to make it deterministic.

    Do not modify any other test in the file.
  </read_first>
