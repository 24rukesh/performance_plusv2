### Phase 6: Waitlist Backend

**Goal**: A visitor can submit their email address and be immediately acknowledged, with the signup stored in Postgres and the product owner notified by email at info@k-innovative.com.
**Depends on**: Phase 5
**Requirements**: WAIT-01, WAIT-02, WAIT-03
**Plans**: 2 plans

Plans:

- [x] 06-01-PLAN.md — Foundations: email-validator dep, .env.example SMTP vars, waitlist table + insert_waitlist_email, WaitlistRequest model, api/email_utils.py SMTP helper
- [x] 06-02-PLAN.md — Wire endpoint + tests: CORSMiddleware, POST /api/waitlist public handler, 4 contract tests (200/422/409/500)

**Details:**

- email-validator==2.3.0 via uv; Pydantic EmailStr validates RFC-compliant format before DB or SMTP call
- UniqueViolation caught outside `with conn:` block (Pitfall 1 avoidance — ensures rollback completes before except runs)
- smtplib STARTTLS on port 587 (Gmail, Zoho, Mailgun compatible); recipient hardcoded to info@k-innovative.com (D-04)
- SMTP failures raise (no try/except in helper) — caller maps to 500 for fail-loudly behavior (D-06)
- DB insert before SMTP send (D-14) — DB commit guaranteed before email dispatch

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| psycopg2-binary over asyncpg | Sync driver matches FastAPI sync def endpoints; no asyncpg wheel complexity | ✓ Good — 14 tests pass with full mock isolation |
| smtplib stdlib only (no sendgrid/mailjet) | Single dependency, covers STARTTLS for major SMTP relays | ✓ Good — zero external email SDK needed |
| Tailwind v4 CSS-first config (@theme block) | v4 silently ignores tailwind.config.ts; @theme in globals.css is the supported path | ✓ Good — avoids silent config file being ignored |
| NEXT_PUBLIC_API_BASE_URL absent in production | Same-origin /api path after Caddy proxying; env var only for local dev override | ✓ Good — no build-time baking risk |
| Island architecture (Server + Client islands) | Next.js Server Components for static sections; single client boundary for WaitlistForm | ✓ Good — production build exits 0, clean tsc |
| handle vs handle_path in Caddyfile | handle preserves prefix (FastAPI needs /api/), handle_path strips it (Streamlit serves at /) | ✓ Good — all 7 deploy config tests pass |
| Postgres healthcheck + depends_on condition | Eliminates init_db() startup race on cold start | ✓ Good — no connection failures at compose up |
| .env* glob in landing/.dockerignore | Blocks .env.local NEXT_PUBLIC_ value from baking into JS bundle | ✓ Good — production URL is clean relative path |
