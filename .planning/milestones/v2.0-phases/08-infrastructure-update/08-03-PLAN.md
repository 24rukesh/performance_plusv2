<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet (HTTPS :443) → Caddy | Untrusted client request crosses here. Caddy terminates TLS, parses HTTP, then dispatches to one of three internal services based on path matcher. |
| Caddy → fastapi:8000 / app:8501 / landing:3000 | Internal Docker bridge network. No TLS between containers — acceptable because the bridge is host-local and not exposed. |
| Caddy admin API (default :2019 on loopback) | Caddy admin API is bound to 127.0.0.1:2019 by default — not reachable from the host network or from other containers because the caddy container does not publish port 2019. No mitigation needed unless `admin :2019` is added explicitly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-11 | Spoofing | /api/* prefix stripping mistake | mitigate | Acceptance criterion enforces `handle /api/*` (preserving prefix) NOT `handle_path /api/*` (stripping it). If `/api/` is stripped, FastAPI receives `/health` instead of `/api/health` — all 5 endpoints return 404, but a custom-mounted unauthenticated root route (none currently exists, but future risk) could be hit instead. Mitigation is the routing semantics themselves, enforced by exact-substring grep in the verify block. (D-07) |
| T-08-12 | Tampering | handle_path /app* glob missing wildcard | mitigate | Acceptance criterion enforces literal `handle_path /app*` AND absence of `handle_path /app/*`. The `/app*` glob matches `/app`, `/app/`, and `/app/anything`. The wrong-looking `/app/*` glob misses bare `/app` requests and they fall through to the catch-all (Next.js), serving the landing page where the user expected the Streamlit app — silent broken behavior. (Pitfall 2 in RESEARCH.md) |
| T-08-13 | Information Disclosure | Open reverse-proxy to arbitrary upstream | accept | The three `reverse_proxy` directives point at fixed Docker service names (`fastapi:8000`, `app:8501`, `landing:3000`) — there is no path-based or user-controlled upstream selection. Caddy cannot be tricked into proxying to an arbitrary upstream (no SSRF vector). |
| T-08-14 | Denial of Service | WebSocket exhaustion on Streamlit upstream | accept | A misbehaving client could open many concurrent Streamlit WebSocket connections via /app and exhaust app:8501. Out of scope for Phase 8 — would require Caddy rate-limiting middleware (not in CONTEXT.md). Synthetic data + demo scale makes this acceptable for now. |
| T-08-15 | Information Disclosure | Caddy admin API exposure | mitigate | Default Caddy admin API binds to 127.0.0.1:2019 inside the container, and the compose `caddy` service publishes only ports 80/443/443-udp (verified from compose.yaml lines 14-17). The Caddyfile must NOT add an `admin` global option directive that re-binds it to 0.0.0.0. Acceptance criterion enforces absence of literal strings `admin off` and `admin :`. |
| T-08-16 | Tampering | Route order regression (catch-all moved above specific routes) | mitigate | Caddy evaluates `handle` blocks in document order; first match wins. If the catch-all `handle { reverse_proxy landing:3000 }` is placed ABOVE `handle /api/*`, then `/api/health` matches the catch-all first and hits the landing page (which 404s on `/api/health`), making all API requests fail. Acceptance criterion enforces document-order: `handle /api/*` line number < `handle_path /app*` line number < `reverse_proxy landing:3000` line number (verified by Python in the verify block). (D-09) |

**No HIGH threats remain unmitigated.** T-08-11, T-08-12, T-08-15, T-08-16 are enforced by mechanical acceptance_criteria checks parsing Caddyfile content.
