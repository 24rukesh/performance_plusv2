<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build context → image layers | `.dockerignore` filters host files before `COPY . .` ships them into the builder stage |
| Builder stage → runner stage | Only artifacts explicitly `COPY --from=builder` cross this boundary; the runner has no source code, no node_modules, no .env files |
| Container process → host | `USER appuser` (UID 1001 non-root) constrains what the container process can do if exploited |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | landing/.dockerignore | mitigate | Exclude `.env*` glob (not just `.env`) so `landing/.env.local` cannot enter the build context. Next.js 16 reads `.env.local` at build time and bakes `NEXT_PUBLIC_*` values into the JS bundle (verified: landing/node_modules/next/dist/docs/01-app/02-guides/self-hosting.md). If `.env.local` reaches `npm run build`, `NEXT_PUBLIC_API_BASE_URL=http://localhost:8000` is hardcoded into the public bundle served by Caddy — breaks waitlist form AND leaks the dev-only origin. Acceptance criterion enforces `^\.env\*$` line. (Pitfall 4 in RESEARCH.md, ASVS V14.) |
| T-08-02 | Elevation of Privilege | landing/Dockerfile runner stage | mitigate | `USER appuser` directive runs the Node server as UID 1001 non-root, consistent with the existing Python Dockerfile pattern (root Dockerfile lines 31-32 and 56-57). Container escape via `node server.js` exploit cannot acquire root capabilities. Acceptance criterion enforces `USER appuser` present and use of Alpine `addgroup`/`adduser` syntax. |
| T-08-03 | Information Disclosure | landing/Dockerfile build args | mitigate | NO `ARG NEXT_PUBLIC_API_BASE_URL`, NO `ENV NEXT_PUBLIC_*`, NO `ENV OPENAI_API_KEY` in the Dockerfile. NEXT_PUBLIC_API_BASE_URL must remain UNSET at build time so the JS bundle uses relative URLs (`""` + `/api/waitlist` = `/api/waitlist`) and Caddy routes them to FastAPI on the same origin. Hardcoding any of these into the image layer leaks via `docker history`. Acceptance criterion enforces `! grep -q 'NEXT_PUBLIC' landing/Dockerfile` and `! grep -q 'OPENAI_API_KEY' landing/Dockerfile`. (D-14 from Phase 7, ASVS V14.) |
| T-08-04 | Tampering | landing/Dockerfile COPY order | mitigate | The exec-form `CMD ["node", "server.js"]` and the strict COPY order (public → mkdir .next → standalone → static) ensure the runtime serves only files traced into `.next/standalone` by Next.js, not arbitrary files from the builder context. Misordering would silently overwrite traced assets. Acceptance criterion uses awk to assert static COPY line number > standalone COPY line number. |
| T-08-05 | Denial of Service | landing/Dockerfile container resource exhaustion | accept | Container resource limits (memory, CPU) are not in Phase 8 scope. Caddy `restart: unless-stopped` (set in Plan 02) provides crash recovery. Acceptable for hackathon/early-SaaS scale; revisit if real traffic warrants it. |

**No HIGH threats remain unmitigated.** All three identified mitigation threats (T-08-01, T-08-02, T-08-03) are enforced by mechanical acceptance_criteria checks.
</threat_model>
