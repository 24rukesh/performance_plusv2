## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → landing/node_modules | Third-party packages (`next`, `tailwindcss`, `@heroicons/react`, transitive deps) are downloaded at install time. Standard MIT/Apache supply chain. |
| Build-time env (`NEXT_PUBLIC_*`) → JS bundle | `NEXT_PUBLIC_API_BASE_URL` is inlined into the compiled bundle. Anything in this var becomes public. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Information Disclosure (env baking) | `NEXT_PUBLIC_API_BASE_URL` in `landing/.env.local` and `landing/.env.example` | mitigate | Only NON-SECRET public URLs are placed behind the `NEXT_PUBLIC_` prefix. The value `http://localhost:8000` exposes nothing (a developer's local FastAPI port). Production guidance in `landing/.env.example` explicitly says "leave UNSET in production" so the build bundle does NOT bake a URL — relative path `/api/waitlist` is served via Caddy same-origin (Phase 8). Never put secrets (API_KEY, OPENAI_API_KEY, SMTP_PASS, DATABASE_URL) under a `NEXT_PUBLIC_` prefix. |
| T-07-06 | Tampering (supply chain) | `npm install` resolves `next`, `tailwindcss`, `react`, `@heroicons/react`, and their transitive deps | accept | All four primary deps are top-N npm packages with active maintainers (Vercel, Tailwind Labs, Meta). `package-lock.json` is committed for reproducible installs. No bespoke registry override; default `registry.npmjs.org` only. Risk equivalent to the rest of the JS ecosystem; standard for v2.0. |
| T-07-07 | Information Disclosure (gitignored .env.local) | `landing/.env.local` | mitigate | The Next.js scaffold's default `landing/.gitignore` already ignores `.env*.local`. Task 1 verifies this guard. Local-dev value (`http://localhost:8000`) is not sensitive even if leaked. |
| T-07-08 | Tampering (Tailwind config drift) | Phase 7 must NOT create `landing/tailwind.config.ts` | mitigate | Pitfall 1: Tailwind v4 silently ignores JS config without `@config` directive; mixing the two creates a hidden source-of-truth split. Task 1 verifies no `tailwind.config.ts` is created; Task 2 places all tokens in `@theme` block in `globals.css`. |
</threat_model>
