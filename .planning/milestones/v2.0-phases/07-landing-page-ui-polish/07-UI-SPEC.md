## Color System

### Rationale for Dark Theme

The landing page uses a dark (near-black) dominant surface. Reasons:
1. The animated demo cards (with green/red/yellow badges) have maximum contrast on dark backgrounds ‚Äî the badge colors were designed against Streamlit's dark sidebar and look washed out on white.
2. AI/ML SaaS marketing convention: dark sites signal technical credibility (Vercel, Linear, Raycast pattern).
3. The Streamlit app uses a light theme ‚Äî the contrast between the two surfaces gives the "Try Demo" CTA a visible mode transition that reinforces the product experience.

### Named Tokens (Tailwind custom colors in `tailwind.config.ts`)

```ts

  brand: {
    bg:        '#0a0a0f',   // dominant ‚Äî page background
    surface:   '#13131a',   // secondary ‚Äî cards, sections
    border:    '#1e1e2e',   // subtle dividers
    muted:     '#6b7280',   // muted text, captions
    text:      '#f0f2f6',   // primary text (matches Streamlit default)
    accent:    '#f63366',   // Streamlit pink ‚Äî reserved for CTAs and highlights
  },
  badge: {
    increase:  '#09ab3b',   // from ui_helpers.py ‚Äî LOCKED
    pause:     '#ff2b2b',   // from ui_helpers.py ‚Äî LOCKED
    decrease:  '#faca2b',   // from ui_helpers.py ‚Äî LOCKED
```

## Spacing Scale

Base grid: 4px. Tailwind default scale maps exactly.

| Token | Tailwind class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `p-1` / `gap-1` | 4px | Icon gaps, badge internal padding |
| sm | `p-2` / `gap-2` | 8px | Compact element spacing, badge pill padding (8px √ó 4px) |
| md | `p-4` / `gap-4` | 16px | Default element spacing, input padding |
| lg | `p-6` / `gap-6` | 24px | Card padding, section-internal spacing |
| xl | `p-8` / `gap-8` | 32px | Layout column gaps |
| 2xl | `p-12` / `gap-12` | 48px | Major section breaks, hero vertical padding |
| 3xl | `p-16` / `gap-16` | 64px | Page-level section padding (top/bottom on each `<section>`) |

**Exceptions:**

| Token | Value | Usage | Justification |
|-------|-------|-------|---------------|
| `4xl` | 96px | Hero section top padding ‚Äî above-fold vertical spacer | Grid-aligned (96 = 4√ó24). Approved: needed for hero visual breathing room above fold. Equivalent to Tailwind `p-24`. |
| ‚Äî | 44px (min-height) | Touch targets for all interactive elements | WCAG 2.5.5 ‚Äî use `min-h-[44px]` on all buttons and links |
| ‚Äî | 48px (height) | Waitlist email input | Comfortable tap target on mobile ‚Äî use `h-12` |

**Streamlit override (not a Tailwind spacing token):** "Back to site" link in Streamlit header uses inline style `font-size: 13px; margin: 0;` ‚Äî this is a Streamlit-specific `st.markdown` HTML override, not part of the landing page spacing scale.

## Typography

### Font Loading (Next.js `app/layout.tsx`)

```tsx
// Use next/font/google for zero-layout-shift loading
import { IBM_Plex_Sans, IBM_Plex_Mono } from 'next/font/google'

const ibmPlexSans = IBM_Plex_Sans({
  subsets: ['latin'],

  variable: '--font-sans',
})
const ibmPlexMono = IBM_Plex_Mono({
  subsets: ['latin'],
  weight: ['400'],
  variable: '--font-mono',

```

Apply both variables to `<html>` element. Set Tailwind `fontFamily.sans` to `['var(--font-sans)', 'system-ui', 'sans-serif']` and `fontFamily.mono` to `['var(--font-mono)', 'monospace']`.

### Type Scale (Landing Page ‚Äî Tailwind classes)

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Display | 48px | `text-5xl` | 600 (`font-semibold`) | 1.1 (`leading-tight`) | Hero product name "Performance Plus" only |
| Heading | 28px | `text-3xl` | 600 (`font-semibold`) | 1.2 (`leading-snug`) | Section headings: "How It Works", "See It In Action", "Why Performance Plus" |
| Sub-heading | 20px | `text-xl` | 600 (`font-semibold`) | 1.3 (`leading-snug`) | Feature card titles, step titles |
| Body | 16px | `text-base` | 400 (`font-normal`) or 600 (`font-semibold`) | 1.5 (`leading-relaxed`) | Hero value prop, feature descriptions, footer copy; semibold variant used for button labels, badge text, and form labels |

**Type scale count: 4 sizes (Display / Heading / Sub-heading / Body). Maximum enforced.**

**Notes on sizes outside this scale:**

- **Badge text (inline style):** Demo animation card badges use `font-size: 12px; font-weight: 600` as an inline style in the `_badge_html()` output ‚Äî this matches `ui_helpers.py` exactly and is a component-level override, NOT a Tailwind class in the landing page type scale.
- **Campaign ID mono labels (demo cards):** Campaign IDs in demo animation cards render as `font-mono` with `text-base` (16px) ‚Äî same size as Body, different font family. No separate size token required.
- **Streamlit "Back to site" link:** Uses `font-size: 13px` as an inline style in an `st.markdown` HTML block in Python. This is a Streamlit-specific override, not a landing page Tailwind class. See Streamlit Polish section.

**Weight rules:**
- Two weights only: 400 (body copy, mono labels) and 600 (headings, button labels, badge text, sub-headings)
- IBM Plex Mono is used ONLY for campaign ID values inside demo animation cards ‚Äî not for general copy
- No italic, no underline except on hover for links
- Hero display text on mobile: scale down to `text-4xl` (36px) at `sm` breakpoint

## Waitlist Form Interaction Contract

**States:**

| State | Input | Button | Message |
|-------|-------|--------|---------|
| Idle | Enabled, empty | "Join Waitlist" ‚Äî enabled | None |
| Typing | Enabled, value | "Join Waitlist" ‚Äî enabled | None |
| Submitting | Disabled | "Joining..." ‚Äî disabled, `opacity-75 cursor-not-allowed` | None |
| Success | Hidden (entire form row hidden) | Hidden | "You're on the waitlist! We'll be in touch." ‚Äî `text-green-400` |
| Duplicate (409) | Re-enabled | Re-enabled | "You're already on the waitlist." ‚Äî `text-brand-muted` |
| Error (500/network) | Re-enabled | Re-enabled | "Something went wrong ‚Äî please try again." ‚Äî `text-red-400` |

**On success:** The form row (`<input>` + `<button>`) is replaced inline with the success message. Use React state: `const [submitted, setSubmitted] = useState(false)`. When `submitted === true`, render the success message div instead of the form row. Form label and section heading remain visible.

**API call spec:**
```ts
fetch(`${process.env.NEXT_PUBLIC_API_BASE_URL ?? ''}/api/waitlist`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email }),
})
```

Local dev: set `NEXT_PUBLIC_API_BASE_URL=http://localhost:8000` in `landing/.env.local`.
Production: leave unset (empty = relative path, Caddy proxies `/api` to FastAPI).
