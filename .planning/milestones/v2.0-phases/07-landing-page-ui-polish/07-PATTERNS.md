**Role:** Client component (React)
**Data flow:** request-response (POST `/api/waitlist` → 200 | 409 | 500)
**Analog (data flow only — no React analog in repo):**

- **State-machine pattern:** `app.py` lines 75-95 (button → try/except → status display) — the closest "Python equivalent" of the React form's submit/success/error states

**API contract** (`api/main.py` lines 76-83) — the exact JSON the form will receive:
```python
@app.post("/api/waitlist", status_code=200)
def waitlist_signup(body: WaitlistRequest):
    signed_up_at = insert_waitlist_email(body.email)
    try:
        send_waitlist_notification(body.email, signed_up_at)
    except Exception as exc:
        raise HTTPException(status_code=500, detail=f"SMTP error: {exc}")
    return {"message": "You're on the waitlist! We'll be in touch."}
```

Confirms: success body is `{"message": "..."}` — but per UI-SPEC, the React form hardcodes the string (D-10) rather than parsing the body. Resolves RESEARCH.md Assumption A2: hardcode the string, switch on `res.status`/`res.ok` only.

**CORS context** (`api/main.py` line 34) — already configured, no preflight blocker:
```python
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])
```

**State-machine analog** (`app.py` lines 75-95) — Python state pattern the React form mirrors:
```python
if st.button("Load Demo Data", type="primary"):
    try:
        web_df, crm_df, merged_df = load_demo_data()
    except FileNotFoundError:
        st.error("Demo data files not found. Run: python data_generator.py")
        st.stop()
    except pd.errors.MergeError:
        st.error(
            "Data error: duplicate session_id detected in CRM data. "
            "Check data/crm_data.csv."
        )
```

- On `res.ok` → `setStatus("success")`; on `res.status === 409` → `setStatus("duplicate")`; on any other branch / `catch` → `setStatus("error")`.
- Error messages are HARDCODED strings from UI-SPEC §Copywriting Contract — never display raw `detail` from the API response (D-12).

**Badge color/label mapping to mirror** (`ui_helpers.py` lines 4-22):
```python
def _badge_html(action: str) -> str:
    colors = {
        "increase": "background:#09ab3b; color:#ffffff;",
        "pause": "background:#ff2b2b; color:#ffffff;",
        "decrease": "background:#faca2b; color:#262730;",
        "insufficient_data": "background:#808495; color:#ffffff;",
    }
    labels = {
        "increase": "INCREASE",
        "pause": "PAUSE",
        "decrease": "DECREASE",
        "insufficient_data": "INSUFFICIENT DATA",
    }
    style = colors.get(action, "background:#808495; color:#ffffff;")
    label = labels.get(action, action.upper())
    return (
        f'<span style="display:inline-block; padding:4px 8px; border-radius:12px; '
        f'font-size:12px; font-weight:600; letter-spacing:0.5px; {style}">{label}</span>'
    )
```

**Pct color mapping to mirror** (`ui_helpers.py` lines 25-31):
```python
def _pct_html(value: int) -> str:
    if value > 0:
        return f'<span style="color:#09ab3b; font-weight:600;">+{value}%</span>'
    elif value < 0:
        return f'<span style="color:#ff2b2b; font-weight:600;">{value}%</span>'
    else:
        return f'<span style="color:#808495; font-weight:600;">0%</span>'
```
