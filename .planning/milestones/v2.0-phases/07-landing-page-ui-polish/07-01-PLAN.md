<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pydantic-validated LLM output → Streamlit HTML render | `result.campaigns[i].semantic_reasoning` is a string produced by gpt-4o Structured Outputs (Pydantic-validated by `CampaignAction` in llm.py) and rendered via `st.write()` (which auto-escapes markdown). The badge HTML is built from a finite, developer-controlled enum (`budget_action` Literal). No user-supplied browser-form input flows into any `unsafe_allow_html` body in this plan. |
| Streamlit branded header HTML constant → browser DOM | `BRANDED_HEADER_HTML` is a static string literal defined in source; no interpolation of any external input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering / Information Disclosure (XSS) | `st.markdown(BRANDED_HEADER_HTML, unsafe_allow_html=True)` in app.py | accept | Header HTML is a hardcoded module-level constant with no string interpolation. No user input touches this path. Verified safe by code reading. |
| T-07-02 | Tampering / Information Disclosure (XSS) | `st.markdown(f"{_badge_html(c.budget_action)}  &nbsp;  {_pct_html(c.percentage_change)}", unsafe_allow_html=True)` inside expander body | mitigate | `_badge_html` accepts only the `budget_action` value which is constrained by Pydantic `Literal["increase","pause","decrease","insufficient_data"]` in llm.py — the function never echoes user input. `_pct_html` accepts an `int` only. Both helpers emit hardcoded HTML scaffolding with style-only interpolation. Per PATTERNS.md §S-2 discipline: no user-supplied strings reach the `unsafe_allow_html` body. |
| T-07-03 | Tampering / Information Disclosure (XSS via semantic_reasoning) | `st.write(c.semantic_reasoning)` inside expander body | mitigate | `st.write()` auto-escapes HTML/markdown; it is NOT wrapped in `unsafe_allow_html=True`. Even if gpt-4o emitted an HTML payload, it would render as escaped text. Defense-in-depth: `semantic_reasoning` is a Pydantic-validated string from a server-side LLM call (not user-controlled). |
| T-07-04 | Information Disclosure (link href) | `<a href="/" target="_self">← Back to site</a>` in branded header | accept | Hardcoded relative path. In local dev (port 8501) this is a no-op refresh; in production Caddy (Phase 8) routes `/` to the Next.js landing page. No sensitive query strings, no user data in URL. |
</threat_model>
