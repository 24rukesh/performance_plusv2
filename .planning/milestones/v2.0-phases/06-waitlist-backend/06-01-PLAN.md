<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI request body | Visitor-supplied email arrives as untrusted JSON before validation |
| FastAPI process → Postgres | Email value crosses into a SQL statement (must be parameterized) |
| FastAPI process → SMTP relay | Email value and timestamp cross into an outbound SMTP envelope (must not enable header injection) |
| Filesystem → process env | SMTP credentials enter the process via env vars; must not be hardcoded or logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | api/models.py WaitlistRequest | mitigate | `EmailStr` validates RFC-compliant format server-side via `email-validator` — rejects malformed addresses, addresses containing `\r\n` (email-header injection), oversized local parts, and missing TLDs before any DB/SMTP call (Task 3 acceptance: WaitlistRequest('not-an-email') raises ValidationError) |
| T-06-02 | Tampering | api/db.py insert_waitlist_email | mitigate | Parameterized SQL with `%s` placeholders — `cur.execute("INSERT ... VALUES (%s, %s) ...", (email, "landing_page"))`. No f-string interpolation. SQL injection impossible (Task 2 acceptance: source contains exact parameterized INSERT statement) |
| T-06-03 | Information Disclosure | SMTP credentials in image layers / docker history | mitigate | All five SMTP vars (`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`) loaded via `os.environ[...]` at call time — no hardcoded secrets in `api/email_utils.py`. `.env.example` ships placeholders only; real `.env` is gitignored (existing pattern from Phase 5 OPENAI_API_KEY / API_KEY) |
| T-06-04 | Denial of Service | Postgres waitlist row explosion via repeated duplicate signups | mitigate | `UNIQUE` constraint on `email` column blocks duplicate rows at the DB layer (D-07, D-09). `psycopg2.errors.UniqueViolation` mapped to 409 — single SQL roundtrip per attempt, no row growth |
| T-06-05 | Denial of Service | psycopg2 connection leak after UniqueViolation | mitigate | `try/finally conn.close()` wraps every DB function (Task 2 — `insert_waitlist_email` follows the same pattern as existing `insert_pending_session`). Catching UniqueViolation OUTSIDE `with conn:` block (Pitfall 1) ensures the context manager rolls back cleanly before the except runs — no aborted-transaction reuse |
| T-06-06 | Tampering | SMTP envelope construction in api/email_utils.py | mitigate | `email` variable is interpolated into `Subject:` and body, but `EmailStr` (T-06-01) has already rejected addresses containing `\r\n`, so SMTP-header injection via the email field is blocked at the model layer before reaching email_utils. `from_addr` is read from env (operator-controlled, not user input). `signed_up_at` is the ISO string from psycopg2 (DB-generated, not user input) |
| T-06-07 | Repudiation | First signup after deployment silently fails on missing SMTP env var | mitigate | `os.environ["SMTP_HOST"]` (bracket access) raises `KeyError` at call time; combined with D-06 (500 on SMTP failure) the operator sees the misconfiguration immediately on first signup rather than later. `.env.example` documents all 5 required vars so the operator has a clear template (Task 1) |
| T-06-08 | Information Disclosure | SMTP failure error message returned to client | accept | D-06 explicitly chooses "fail loudly" — 500 with `detail=f"SMTP error: {exc}"` is acceptable for single-tenant MVP. Owner-only deployment, no public listing of error details. Future hardening: scrub `{exc}` repr in multi-tenant release |
</threat_model>
