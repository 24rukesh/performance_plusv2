<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted client → /api/* endpoints | Every request body is untrusted JSON; FastAPI + Pydantic validates the shape; verify_api_key dependency gates protected endpoints |
| /api/analyze endpoint → gpt-4o | Untrusted CSV text built from user-supplied sales_notes is sent to OpenAI in the user message — bounded by Pydantic field types and prompt isolation in llm.py |
| /api/analyze endpoint → Postgres | Result persistence uses parameterized SQL (in api/db.py from Plan 01); api/main.py never constructs SQL strings |
| /api/health (no auth) → caller | Returns only static service metadata — no secrets, no DB state, no user data |
