<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Docker images | `docker build` runs user-supplied Dockerfile; the only secret-bearing input is `.env` (mounted via env_file:, not baked into the image) |
| Public internet → Caddy | Caddy on host ports 80/443 is the only externally reachable surface — fastapi and postgres are on the internal bridge network with no exposed ports |
| fastapi container → postgres container | Both on the internal docker bridge network; postgres password supplied via env var, not in image |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure (secret in image layer) | Dockerfile | mitigate | No ARG or ENV instruction in Dockerfile sets any secret value. `.env` is mounted at runtime via compose.yaml `env_file: .env`, never COPYed into the image (verified by absence of `COPY .env` in Dockerfile). |
| T-05-02-02 | Information Disclosure (hardcoded postgres password) | compose.yaml | mitigate | postgres password sourced as `${POSTGRES_PASSWORD}` from `.env`. Acceptance grep gate asserts no quoted-literal password in compose.yaml. |
| T-05-02-03 | Elevation of Privilege (root in container) | Dockerfile streamlit + api stages | mitigate | Both runtime stages run as appuser (UID 1001) via `USER appuser`. Verified by `grep -c "USER appuser"` returning 2. |
| T-05-02-04 | Information Disclosure (postgres exposed on public internet) | compose.yaml postgres service | mitigate | postgres service has NO `ports:` key — only reachable via internal docker bridge network. Acceptance gate explicitly asserts `'ports' not in c['services']['postgres']`. |
| T-05-02-05 | Spoofing (wrong Dockerfile target builds wrong service — Pitfall 1) | compose.yaml app service | mitigate | `app` service explicitly declares `target: streamlit`. Without this, after the multi-stage rewrite Docker would default to the last stage and the "Streamlit" container would silently start uvicorn instead. Acceptance gate asserts `c['services']['app']['build']['target'] == 'streamlit'`. |
| T-05-02-06 | Information Disclosure (postgres volume on host) | compose.yaml postgres_data volume | accept | Named volume managed by Docker on the host filesystem. Standard practice; protected by host filesystem permissions. Not encrypted at rest — acceptable for synthetic/mock data per Phase 5 scope. |
| T-05-02-07 | Denial of Service (uvicorn workers / worker count tuning) | Dockerfile api stage CMD | accept | Default uvicorn single-worker is used (no `--workers` flag). For v2.0 demo scale this is sufficient. Multi-worker / gunicorn-uvicorn worker class deferred to v3+. |
</threat_model>
