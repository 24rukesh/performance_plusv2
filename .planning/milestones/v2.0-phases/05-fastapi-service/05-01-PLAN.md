## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering (SQL injection) | api/db.py — all 4 SQL statements | mitigate | All cur.execute() calls use psycopg2 `%s` parameterized placeholders. The grep gate in Task 3 acceptance_criteria asserts zero f-string or .format() SQL construction in non-comment lines. |
| T-05-01-02 | Information Disclosure (env var leakage) | .env.example | mitigate | .env.example contains only placeholder values (`your-api-key-here`, `password`, `your-postgres-password-here`) — no real secrets. Real `.env` already gitignored from Phase 4. |
| T-05-01-03 | Information Disclosure (psycopg2 stack traces with DATABASE_URL in error message) | api/db.py get_conn() | accept | Phase 5 scope: single-tenant demo. psycopg2.OperationalError on connection failure may surface DSN in logs but not in HTTP responses — api/main.py (Plan 03) wraps the call and FastAPI default error handler does not include env vars in response body. Acceptable per V8 ASVS Level 1 for synthetic data. |
| T-05-01-04 | Denial of Service (connection exhaustion) | api/db.py — per-call connections without pool | mitigate | try/finally conn.close() pattern in every function (Pitfall 3) closes each connection deterministically. For v2.0 demo scale (low concurrency) this is sufficient. Connection pool deferred to v3+. |
| T-05-01-05 | Information Disclosure (lock file leaks transitive dep tree) | uv.lock | accept | Standard practice — uv.lock is committed to git for reproducibility (per CLAUDE.md "What NOT to Use" section). Discloses dep tree, not secrets. |
