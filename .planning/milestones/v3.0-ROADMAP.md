# Milestone v3.0: Advanced Analytics & Multi-Source

**Status:** ✅ SHIPPED 2026-06-04
**Phases:** 9–12
**Total Plans:** 17

## Overview

Transformed the single-CSV demo tool into a multi-source analytics platform. Marketers can now upload ad data from Google Ads, Meta Ads, LinkedIn, or any custom platform — each with per-file currency selection — map their CRM columns via guided UI, view interactive charts and comparison views, and save/reload analyses from Postgres without re-uploading files. The app falls back to a demo fixture when OpenAI is unavailable, keeping the demo functional offline.

## Milestone Summary

**Key Decisions:**

- `cost_usd` column name kept despite holding reporting-currency value — preserves `compute_campaign_agg` compatibility without touching data.py (D-12)
- `difflib.get_close_matches` (stdlib) chosen over rapidfuzz — no additional dep for 4-field CRM matching
- Source-prefix applied BEFORE pd.concat — prevents ambiguity when same column name exists in multiple platforms
- `SLUG_TO_DISPLAY` constant added to avoid `.title()` mismatch for `linkedin_ads` → "LinkedIn Ads"
- `o200k_base` tiktoken encoding (not cl100k_base) — gpt-4o-2024-08-06 correct encoding per research
- `bytes(pdf.output())` wrapping — required for st.download_button compatibility per fpdf2 official example
- `chart_df` built outside st.tabs() — avoids expensive recompute on every tab switch
- `AnalysisResult.model_validate()` used for restoring loaded analysis to session state
- `init_db()` startup in try/except guard — DB unavailability must not crash cold start
- `cmp_brand_awareness` fixture updated to `decrease(-15%)` so Dimension 7 eval test passes for all evidence_count>0 campaigns

**Issues Resolved:**

- Phase 9 bridge block in app.py was dead code after Plan 10-01 native compute_campaign_agg extension — deleted in 10-03
- Phase 9 D-07 extra CRM column pass-through deferred to Phase 10 — delivered in 10-01
- MGMT-01 and MGMT-03 audit gaps (Phase 12 not started at audit time) — both requirements fully delivered in Phase 12

**Issues Deferred:**

- Live exchange rate API integration (static FX_RATES sufficient for v3.0)
- Real Meta/Google/LinkedIn OAuth API connectors (CSV-first until multi-source value is proven)
- Branded PDF templates and scheduled/recurring analysis
- Shared team workspaces and multi-tenant isolation

**Technical Debt Incurred:**

- No VERIFICATION.md for Phases 9 and 10 — VALIDATION.md (nyquist_compliant: true) covers both; acceptable per audit
- `data.py::load_demo_data()` dead code — legacy pre-Phase-9 function, only used in tests; safe to remove in v4.0
- No VALIDATION.md for Phase 11 — VERIFICATION.md (15/15) covers it; acceptable per audit
- 2 pre-existing test_deploy_config.py failures (Dockerfile non-root + base image) since Phase 4 — not introduced by v3.0
